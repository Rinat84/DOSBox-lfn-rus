:CONFIG_FULLSCREEN
Запускает DOSBox сразу в полноэкранном режиме. (Для отключения нажмите ALT_ENTER)
.
:CONFIG_FULLDOUBLE
Использовать двойную буферизацию при полноэкранном режиме. Может уменьшить мерцание экрана, но возможно приведёт к замедлению DOSBox.
.
:CONFIG_FULLRESOLUTION
Какое разрешение использовать для полного экрана: оригинальное или фиксированное значение (например, 1024x768).
  Использование собственного разрешения монитора с aspect=true может дать наилучшие результаты.
  Если вы получаете маленькое окошко на большом экране, попробуйте метод вывода отличный от surface.
.
:CONFIG_WINDOWRESOLUTION
Масштабировать окно к этому размеру, ECЛИ устройство вывода поддерживает аппаратное масштабирование.
  (Режим вывода output=surface не поддерживается!)
.
:CONFIG_OUTPUT
Какую видеосистему использовать для вывода.
.
:CONFIG_AUTOLOCK
Мышь автоматически захватывается, если вы нажмете на экран.(Нажмите CTRL-F10, чтобы разблокировать)
.
:CONFIG_SENSITIVITY
Чувствительность мыши.
.
:CONFIG_WAITONERROR
Ждать перед закрытием консоли, если в DOSBox происходит ошибка.
.
:CONFIG_PRIORITY
Уровни приоритета процесса DOSBox. Второй параметр (после запятой) используется, когда DOSBox не активен или свернут.
  Значение 'pause' поддерживается только для второго параметра.

:CONFIG_MAPPERFILE
Файл, используемый для загрузки/сохранения раскладки клавиш. Resetmapper работает только со значением по умолчанию.

:CONFIG_MEMSIZE
Объем памяти для DOSBox в мегабайтах.
  Значение по умолчанию - 30, но может быть установлено другое значение например 16, чтобы избежать проблем с определенными играми,
  хотя для нескольких игр может потребоваться более высокий объем.
  Увеличение этого значения в общем случае не повышает быстродействие.

Масштабирование используется для увеличения/улучшения режима низкого разрешения.

:CONFIG_CORE
Ядро процессора используется для эмуляции. 'auto' переключает на динамический, если он доступный и соответствующий.

:CONFIG_CPUTYPE
Тип процессора используется для эмуляции. 'auto' выбирает самый быстрый вариант.
.
:CONFIG_CYCLES
Количество инструкций, которое DOSBox пытается эмулировать каждую  миллисекунду.
Слишком высокое значение этого параметра приводит к отключению звука и задержкам.
Циклы могут быть установлены тремя способами:
  'auto'          Пытается подобрать нужное значение игре. Обычно работает, но для некоторых игр создает проблемы.
  'fixed #число'  устанавливает фиксированное число циклов. Это обычно нужно, если 'auto' не работает (Например: fixed 4000).
  'max'           будет выделять столько циклов, сколько может обрабатывать ваш компьютер.
.
:CONFIG_CYCLEUP
Количество циклов для уменьшения/увеличения с помощью клавиш.(CTRL-F11/CTRL-F12)

:CONFIG_CYCLEDOWN

:CONFIG_BLOCKSIZE

:CONFIG_PREBUFFER

:CONFIG_MIDICONFIG

  которое вы хотите использовать (найдите id/name с помощью mixer/listmidi).
  Или, в случае с Coreaudio, здесь вы можете указать банк инструментов (MIDI SoundFont).
  При использовании Roland MT-32 rev.0 в качестве устройства вывода midi, для некоторых игр может потребоваться задержка, чтобы предотвратить проблемы с переполнением буфера.

  Подробнее см. README / Manual.

:PROGRAM_INTRO_CDROM
[2J[32;1mКак смонтировать Реальный/Виртуальный CD-ROM привод в DOSBox:[0m
DOSBox поддерживает эмуляцию CD-ROM на нескольких уровнях.

[33mОсновной[0m уровень работает со всеми дисками CD-ROM и обычными каталогами.
Он устанавливает MSCDEX и помечает файлы атрибутом "только для чтения".
Обычно этого достаточно для большинства игр:
[34;1mmount d [0;31mD:\[34;1m -t cdrom[0m   или   [34;1mmount d C:\example -t cdrom[0m
Если это не срабатывает, вы можете указать DOSBox метку CD-ROM:
[34;1mmount d C:\example -t cdrom -label МЕТКА[0m

[33mСледующий[0m уровень добавляет кое-что из низкоуровневой поддержки.
Поэтому он работает только с приводами CD-ROM:
[34;1mmount d [0;31mD:\[34;1m -t cdrom -usecd [33m0[0m

[33mПоследний[0m уровень поддержки зависит от вашей операционной системы:
Для [1mWindows 2000[0m, [1mWindows XP[0m и [1mLinux[0m:
[34;1mmount d [0;31mD:\[34;1m -t cdrom -usecd [33m0 [34m-ioctl[0m
Для [1mWindows 9x[0m с установленным ASPI layer:
[34;1mmount d [0;31mD:\[34;1m -t cdrom -usecd [33m0 [34m-aspi[0m

Замените [0;31mD:\[0m на букву вашего CD-ROM.
Замените [33;1m0[0m в [34;1m-usecd [33m0[0m на номер, полученный для вашего CD-ROM командой:
[34;1mmount -cd[0m
